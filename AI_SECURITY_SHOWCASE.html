🔑 OpenAI API key exposed in client-side JavaScript

Critical
The OpenAI API key is hardcoded in app.js loaded by the browser. Anyone visiting the site can view source, extract the key, and run up charges on your account — or exfiltrate your prompt templates and customer data.
Fix: Move all API calls to a server-side proxy. Rotate the compromised key immediately. Budget: 4 hours.

🛡️ No prompt injection protection on customer-facing chatbot

Critical
The public chatbot accepts raw user input directly into the GPT prompt with no sanitisation. An attacker can use prompt injection to bypass system instructions, leak internal data, or make the bot generate harmful content under your brand.
Fix: Add input sanitisation, system prompt hardening, and output filtering. Budget: 3 hours.

🔓 API keys stored in GitHub repository (public repo)

High
3 API keys found in commit history on a public GitHub repo. Even if removed from current code, they're accessible in git history. These keys have likely been scanned by automated tools.
Fix: Rotate all exposed keys. Remove from repo using BFG Repo-Cleaner or git filter-branch. Add .env to .gitignore. Budget: 2 hours.

📊 No rate limiting on AI API endpoints

High
The API proxy accepts unlimited requests. A malicious actor or buggy integration could trigger thousands of GPT API calls in minutes, resulting in a surprise A$500+ bill.
Fix: Implement token bucket rate limiting. Add usage alerts at 80% and 100% of daily budget. Budget: 2 hours.

🔐 No encryption for data at rest in automation database

High
Customer data stored in plaintext in Supabase. If the database is compromised, all historical customer interactions are exposed.
Fix: Enable Supabase Vault for secrets. Add column-level encryption for PII. Budget: 3 hours.

📝 No error logging or monitoring for automation failures

Medium
When the n8n workflow fails (API timeout, bad data, rate limit), there's no alert. Failures are discovered when customers complain — sometimes days later.
Fix: Add error webhook → Slack alert. Add Sentry or simple error log DB. Budget: 1.5 hours.

🔄 No API key rotation policy

Medium
Keys haven't been rotated in 12+ months. If a key was ever accidentally exposed (Slack message, support ticket screenshot), it's still valid.
Fix: Establish quarterly key rotation. Document in onboarding README. Budget: 1 hour + ongoing.

🌐 CORS misconfiguration on API proxy

Medium
CORS allows requests from any origin (*). While not directly exploitable for this use case, it unnecessarily widens the attack surface.
Fix: Restrict CORS to your specific domain(s). Budget: 30 minutes.

📋 No access audit log for admin actions

Medium
When the system is modified (prompt changed, workflow updated, API key added), there's no record of who did what and when.
Fix: Add simple audit log table. Log user, action, timestamp. Budget: 1.5 hours.

📚 Documentation references outdated API versions

Low
README references deprecated endpoints. Won't break anything today, but could confuse future maintainers.
Fix: Update documentation to current API versions. Budget: 30 minutes.

🔧 Hardcoded environment values in workflow config

Low
Timezone and default language are hardcoded instead of using environment variables. Inconvenient for international deployments.
Fix: Move to environment variables. Budget: 30 minutes.
