## 3. PROPOSAL TEMPLATES

### Template A: The "Pain-First" Proposal (for clients describing a specific problem)

```
Hi {client_name},

I read your job post about {specific problem they described}. 
This is exactly the kind of workflow I specialise in.

**The core issue I see:**
{1-sentence diagnosis — show you understand the problem}

**How I'd solve it:**
1. {Step 1 — specific tool/approach}
2. {Step 2}
3. {Step 3}

**Similar project I delivered:**
I recently built {similar_project} for a {industry} client. 
Result: {specific number} (see portfolio item #X).

**Timeline:** {X} business days
**Fixed price:** A${amount}

I've attached a screenshot of a similar workflow I built. 
Happy to jump on a 10-min call to walk through the approach.

Best,
{your_name}
```

### Template B: The "Better Way" Proposal (for vague job posts)

```
Hi {client_name},

I noticed you're looking for {broad_description}. Most people approach 
this with {common_but_wrong_approach}, but there's a better way.

**The problem with {common_approach}:**
{1-2 sentences on why it breaks or doesn't scale}

**What I'd recommend instead:**
{your_better_approach} using {specific_tool}.

This approach:
✅ Handles {edge_case} that {common_approach} misses
✅ Costs {X}% less to run per month
✅ Can scale to {Y} volume without breaking

**Fixed price:** A${amount} | **Timeline:** {X} days

I've done this exact setup for {number} clients. Happy to share 
a walkthrough video of it in action.

{your_name}
```

### Template C: The "Questions-First" Proposal (for complex/ambiguous posts)

```
Hi {client_name},

This looks like an interesting project. Before I send a proposal, 
I have a few clarifying questions to make sure I scope this right:

1. {Question about volume/scale}
2. {Question about existing tools/stack}
3. {Question about timeline/urgency}

Once I understand these, I can send you a precise fixed-price quote 
(usually A${range_low}–A${range_high} for this type of work).

In the meantime, here's a similar project I delivered: 
{1-line case study with result}.

{your_name}
```

### Proposal Rules

- **Always bid fixed-price** for automation projects. Hourly makes clients nervous about scope.
- **Include at least 1 question** in every proposal. It triples response rate.
- **Mention a specific portfolio item** by number. Shows you read the brief.
- **Never use "Dear Sir/Madam"** — use their name. If no name, use "Hi there".
- **Keep it under 200 words.** Long proposals don't get read.

## 4. JSS PROTECTION STRATEGY

**JSS = Job Success Score.** One bad review tanks it. Protect aggressively.

### Pre-Contract

- [ ] **Screen the client:** Check their hire rate, average rating given, and review history. If they've given <4 stars to anyone, skip.
- [ ] **Fixed scope only:** No open-ended "build me an automation." Define exact deliverables.
- [ ] **Milestone payments:** Never 100% on completion. Structure: 30% upfront / 40% mid-project / 30% on delivery.

### During Project

- [ ] **Daily updates (even one-liners):** "Today I built the Slack integration — testing tomorrow." Silence makes clients nervous.
- [ ] **Screen recordings:** Loom (free) recordings of progress. Shows work happening.
- [ ] **Flag delays BEFORE the deadline:** "Running 1 day behind because {reason}. New ETA: {date}. Here's what's done so far."
- [ ] **Over-deliver on one small thing:** Free bonus (extra dashboard view, documentation PDF, 5-min tutorial video).

### Risk Detection (pull the plug early)

**Red flags that predict a bad review:**
- Client asks for "just one more thing" 3+ times before mid-project
- Client disputes milestone without clear reason
- Client goes silent for 3+ days during review phase
- Client compares you to "my last developer who charged less"

**Response:** "I want to make sure we're aligned. Let's pause new work and confirm the current scope and timeline. Here's what's done and what remains."

### If a Bad Review Happens

1. **Respond publicly:** Professional, factual, no emotion. "We disagreed on {specific point}. I offered {solution}. Happy to resolve this directly."
2. **Refund if necessary:** A$500 refund is cheaper than a 4.2 JSS.
3. **Bury it with good reviews:** 10 five-star reviews drown one bad one.
4. **Top-Rated perk:** Once Top-Rated, you can remove one review per quarter.

## 8. RED FLAGS & SCAM AVOIDANCE

### Immediate Skip (Don't Even Bid)

- "Need this done TODAY" + new account with no hire history
- "Will pay after seeing results" or "test project first"
- Client asks to communicate outside Upwork before contract starts
- Client shares login credentials before contract (liability risk)
- "I have a great idea, just need someone to build it" (vague = scope creep)
- Budget is "$10-30" for something that's clearly a $500+ project

### Contract Protection

1. **Never start work without an active Upwork contract.** No exceptions.
2. **All communication stays on Upwork** until contract starts. Then move to email/Slack.
3. **Milestone-funded only** for fixed-price projects. Escrow protects both sides.
4. **Track time with Upwork Desktop App** for hourly contracts (automatic payment protection).
5. **Get scope confirmed in Upwork messages** before starting: "Just to confirm, the scope is: {list}. Timeline: {date}. Deliverables: {list}."

### Payment Terms

- Fixed-price: 30% upfront milestone → 40% mid-project → 30% on delivery
- Hourly: Upwork's weekly billing cycle (automatic — no action needed)
- Never accept PayPal/Venmo/bank transfer — violates Upwork ToS and loses payment protection
